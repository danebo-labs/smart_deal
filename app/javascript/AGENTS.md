# JavaScript Rules

## Frontend Architecture

* Prefer Stimulus controllers over custom framework patterns.
* Prefer Turbo behavior over client-side routing.
* Avoid SPA-like state management.
* Avoid unnecessary frontend state.
* Keep JavaScript small, focused, and deterministic.

## UX And Performance

* Optimize for mobile-first interaction.
* Keep perceived responsiveness high.
* Avoid polling unless necessary.
* Avoid unnecessary Turbo broadcasts.
* Design for low light, gloves, unreliable connectivity, and small screens.
